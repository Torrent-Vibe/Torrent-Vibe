# Sparkle Updater Migration & Release Pipeline Redesign

Date: 2026-07-15
Repo: `Torrent-Vibe/Torrent-Vibe`
Reference implementation: `/Users/innei/git/trade` (kansoku, `app/desktop`)

## Goal

Replace the custom three-layer update system (renderer hot-update + mainHash
manifest signing + electron-updater) with:

- **macOS**: native Sparkle.framework updates, ported from the kansoku desktop app
- **Windows/Linux**: electron-updater over electron-builder's native `latest.yml`
- A semver-tag-driven GitHub Release pipeline ported from kansoku
- electron-forge fully replaced by electron-builder
- electron-vite fully replaced by tsdown (main/preload) + existing plain Vite (renderer)

Vite is already at 8.0.16 with electron-vite 5 resolving against it — no Vite
work needed beyond removing electron-vite itself.

## Decisions (confirmed with user)

1. **Hybrid platform strategy**: Sparkle on macOS; electron-updater stays for
   Windows/Linux. The custom renderer hot-update system is removed entirely.
2. **Full electron-builder migration** for all platforms; forge is removed.
3. **Semver versioning**: `desktop-vX.Y.Z` tags auto-created when
   `package.json` version changes on `main`; CHANGELOG section mandatory.
4. **macOS arm64 only** (same as kansoku). Intel builds are dropped.
5. **Web release job is kept** as its own workflow; electron-vite is dropped.
6. **Historical cleanup**: after the new pipeline works, delete all legacy
   date-based releases and tags (`v20*`).

## 1. macOS updater — Sparkle bridge

### Native module

Copy `app/desktop/native/sparkle-bridge/` from kansoku into
`native/sparkle-bridge/`:

- `src/sparkle_bridge.mm` — Objective-C++ N-API addon wrapping SPUUpdater
- `binding.gyp`
- `scripts/fetch-sparkle.sh` — downloads and verifies the pinned
  Sparkle.framework tarball into `vendor/`
- `package.json` — `build` = fetch-sparkle + node-gyp rebuild against the
  Electron headers version used by this repo (electron 42.4.0, arm64)
- Add a `build.sh` wrapper so the existing `native/build-all.sh` convention
  picks it up (macOS-only guard inside: skip on win/linux)

### Main-process updater

New `layer/main/src/updater/` ported from kansoku `src/updater/`:

- `updater.ts` — check throttling, GitHub latest-release lookup, Sparkle
  orchestration, persisted last-check state
- `sparkle.ts` — bridge loader (Info.plist keys with hardcoded fallback for
  feed URL and placeholder public key)
- `status.ts` — updater UI status store
- IPC rewritten in this repo's `electron-ipc-decorator` style, exposing:
  check-for-update, status subscription, install-and-restart

Constants:

- Feed URL: `https://github.com/Torrent-Vibe/Torrent-Vibe/releases/latest/download/appcast.xml`
- Public key placeholder: `SPARKLE_ED_PUBLIC_KEY_PLACEHOLDER` (CI injects real key)

### Removals

- `layer/main/src/services/update-service.ts`
- `layer/main/src/services/github-update-service.ts`
- `layer/main/src/services/bridge-service.ts`
- `layer/main/src/manager/update-manager.ts`
- `layer/main/src/types/update-manifest.ts`, `layer/main/src/config/update-config.ts`
- `scripts/generate-update-signing-keypair.js`, `scripts/write-main-hash-dev.js`
- `mainHash` field in root `package.json`; `predev:electron` hash step
- `resources/security/` update signing keys; `resources/app-update.yml` (regenerated by electron-builder)
- All renderer hot-update UI/IPC; renderer update UI rewired to the new
  updater IPC (check / progress / restart-to-install) and to electron-updater
  events on win/linux

## 2. Windows/Linux — electron-updater

`app-auto-updater.ts` is kept but reduced: initialized only on win32/linux;
darwin uses the Sparkle path. electron-builder generates `latest.yml` /
`latest-linux.yml` natively, so the custom
`generate-app-build-manifest.js` / metadata scripts are deleted.

- Windows target: **NSIS x64** (replaces Squirrel; AppX discontinued)
- Linux target: **AppImage x64**

## 3. Packaging — electron-builder replaces forge

New `electron-builder.yml` replacing `forge.config.mts` (16K). Key contents:

- `appId`, `productName: Torrent Vibe`, icons from existing resources
- `files`: `dist/main/**`, `dist/preload/**`, native module release builds,
  `package.json`; `extraResources` for locales/resources as currently packaged
- mac: `dmg` + `zip`, `arch: [arm64]`, `identity: null`,
  `afterPack: scripts/afterPack.cjs` (ad-hoc codesign, ported from kansoku),
  `extendInfo`: `SUFeedURL`, `SUPublicEDKey` placeholder,
  `SUEnableInstallerLauncherService: false`, `SUScheduledCheckInterval: 3600`
- mac `extraFiles`: `native/sparkle-bridge/vendor/Sparkle.framework` →
  `Frameworks/Sparkle.framework`
- `asarUnpack` for native `.node` modules
- `dmg.writeUpdateInfo: false`, `zip.writeUpdateInfo: false` (mac updates ship
  via Sparkle; win/linux blockmaps and yml remain enabled)
- Electron fuses: applied inside `afterPack` via `@electron/fuses`
  (electron-builder has no first-class fuses support; port flag set from
  `forge.config.mts`)
- Artifact naming preserved: `Torrent.Vibe-<version>-<platform>-<arch>.<ext>`
  via `artifactName` (replaces forge rename hook)

Remove all `@electron-forge/*` deps, `@pengx17/electron-forge-maker-appimage`,
`electron-squirrel-startup`, `forge.config.mts`. Add `electron-builder`,
`tsdown`.

## 4. Build tooling — tsdown replaces electron-vite

- `tsdown.config.ts` (root):
  - main: entry `layer/main/src/index.ts` → `dist/main`, ESM, platform node;
    alias `~` → `layer/main/src`, `@locales` → `locales`; define `ELECTRON`,
    `__BUILD_TIME__`; `neverBundle`: electron, electron-liquid-glass, sqlite3,
    bindings, node-gyp-build, bufferutil, utf-8-validate (audit against
    current `MAIN_PROCESS_EXTERNALS` during implementation)
  - preload: entry `layer/main/preload/index.js` → `dist/preload`, CJS
- Renderer: unchanged — plain Vite 8 via existing `vite.config.ts`
  (`ELECTRON: true`, `__WEB_UI__: false` electron-renderer build kept as-is,
  including the multi-entry `index/mini/panel` HTML inputs and the
  platform-specific-import + cleanup plugins, which move from
  `electron.vite.config.ts` into a small renderer-electron Vite config)
- `scripts/dev-electron.mjs` ported from kansoku `scripts/dev.mjs`:
  Vite dev server for renderer + tsdown watch for main/preload + electron
  relaunch on main rebuild
- Delete `electron.vite.config.ts` and the `electron-vite` dependency;
  rewrite `dev:electron`, `build:electron`, `start:electron` scripts

## 5. CI pipelines

### `desktop-tag.yml` (ported verbatim, paths adjusted)

Trigger: push to `main` touching root `package.json`. Reads `version`, tags
`desktop-vX.Y.Z` if absent, dispatches `desktop-release.yml` (GITHUB_TOKEN tag
pushes don't trigger workflows).

### `desktop-release.yml`

Trigger: `desktop-v*` tag push + `workflow_dispatch(tag)`. Three jobs, all
publishing into one GitHub Release:

- **mac (macos-14)**: version-vs-tag consistency check; Sparkle secrets
  assertion; CHANGELOG `## X.Y.Z` section extraction (fail fast);
  typecheck; build natives; tsdown + vite build; electron-builder dmg+zip
  arm64; ad-hoc signature verification of both artifacts; fetch previous
  appcast + last 2 release zips (filtered to `desktop-v*` tags) as delta base;
  `generate_appcast` with EdDSA key on a RAM disk;
  `.github/scripts/fix-appcast-enclosure-urls.mjs` (ported) repoints old
  enclosures at their own tags; `gh release create` with dmg, zip,
  appcast.xml, deltas, CHANGELOG notes
- **windows (windows-latest)**: build + electron-builder NSIS; upload exe +
  `latest.yml` + blockmap to the same release
- **linux (ubuntu-latest)**: build + AppImage; upload AppImage +
  `latest-linux.yml`

Release creation: mac job creates the release; win/linux jobs
`gh release upload` after it exists (mac job is a dependency or upload steps
retry-wait on the release).

Pinned env: `SPARKLE_VERSION`, `SPARKLE_TARBALL_SHA256`,
`REPO_SLUG: Torrent-Vibe/Torrent-Vibe`.

### `web-release.yml`

The current `build-web` job extracted unchanged into its own
workflow_dispatch-triggered workflow producing `qb-client-webui-web.tar.gz`.

### Removals

`build.yml`, `renderer-release.yml`.

### New repo secrets (manual, by user)

`SPARKLE_ED_PUBLIC_KEY`, `SPARKLE_ED_PRIVATE_KEY` — generated with Sparkle's
`generate_keys`; private key never touches the repo.

## 6. Versioning & changelog

- Root `package.json` `version` becomes the real app version, starting at
  `1.0.0`
- Root `CHANGELOG.md` with a mandatory `## X.Y.Z` section per release;
  release job fails without it

## 7. Legacy cleanup (after first successful new-pipeline release)

Delete all legacy date-based releases and their tags (`v20*` pattern) via
`gh release delete --cleanup-tag`. Explicitly requested by user; destructive
and irreversible — executed only after the first `desktop-v*` release is
verified working, and enumerated with a dry-run list first.

## Risks

1. **No migration path for existing installs**: old apps poll date-tag
   releases with electron-updater metadata that will stop being published;
   deleting legacy releases also removes their assets. Existing users must
   manually download the first `desktop-v*` build once.
2. **tsdown externals**: the main process bundles differently than
   electron-vite/rollup; the `neverBundle` list must be audited against
   runtime `require`s of native modules during implementation.
3. **Fuses via afterPack**: flipping fuses after packaging must happen before
   ad-hoc signing in the same hook; order inside `afterPack.cjs` matters.
4. **electron-updater Squirrel→NSIS switch**: any existing Windows installs
   cannot auto-migrate installer format; same manual-reinstall caveat as mac.

- Windows/Linux packages build in CI; electron-updater dev smoke via

- Release dry-run script (port kansoku `release-dry-run.sh`) validates tag /
